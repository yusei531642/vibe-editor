TITLE: [bug] sync: TeamOrchestrationState.teamReports が TS 側に未定義 (team_state.rs:215 と shared.ts のズレ)
LABELS: bug,rust,javascript

## 概要 / Summary

Rust 側 `TeamOrchestrationState` には `team_reports: Vec<TeamReportSnapshot>` が #572 で導入され `team_state_read` の戻り値にも乗っているが、TypeScript 側 `src/types/shared.ts` の `TeamOrchestrationState` interface に対応フィールドが未定義のため、renderer 側で silent drop されている。`TeamReportSnapshot` / `TeamReportFinding` の TS 投影自体も無く、Canvas / Dashboard で team_report を可視化できない。

## 根本原因 / Root cause

- `src-tauri/src/commands/team_state.rs:128-215` で `TeamOrchestrationState` 構造体に `pub team_reports: Vec<TeamReportSnapshot>` が定義されており、`team_state_read` の戻り値 JSON にもキー `teamReports` (serde camelCase) が含まれる。
- `src/types/shared.ts:724-737` の TS interface `TeamOrchestrationState` には対応フィールドが無い。型上 silent に欠落。
- `TeamReportSnapshot` / `TeamReportFinding` の TS 投影が無いため、renderer 側で型安全に取り扱えず、`(state as any).teamReports` のような escape hatch すら無い (= 結果的に未使用)。
- 結果として、worker が `team_report` で送った findings を Leader UI / Canvas のダッシュボードに表示できない。

## 再現手順 / Repro

1. worker に role を割り当て、`team_report({ findings: [...] })` を 1 件以上送る。
2. Rust 側 disk file (`~/.vibe-editor/team-state/<key>/<team>.json`) には `teamReports` が永続化される。
3. renderer 側で `team_state_read` を invoke しても、TS interface に該当フィールドが無いため `state.teamReports` を読み取れず、Canvas Dashboard 等で報告内容を可視化できない。

## 影響範囲 / Impact

- **severity: CRITICAL (sync drift / silent feature loss)**
- 直近 #572 で追加された team_report 機能が UI 上に流れず、worker からの構造化レポートが完全に死んでいる。
- 型システムの保護も外れているため、将来同様のフィールドが追加されたときも気付きにくい (再発しやすい)。
- TS / Rust 間の型シンク同期は CLAUDE.md の根幹原則 (`src/types/shared.ts` を共有 source of truth にする) に直接反するため、CI / lint レベルで防止する仕組みも必要。

## 修正方針 / Fix proposal

### 案1 (推奨): `shared.ts` に TeamReportSnapshot / TeamReportFinding interface を追加し teamReports を生やす

1. `src/types/shared.ts` に Rust 側 `TeamReportSnapshot` / `TeamReportFinding` を 1:1 で投影:
   ```ts
   export interface TeamReportFinding {
     id: string;
     severity: 'critical' | 'high' | 'medium' | 'low' | 'info';
     summary: string;
     description?: string;
     filePath?: string;
     line?: number;
     evidence?: string;
     recommendation?: string;
   }

   export interface TeamReportSnapshot {
     reportId: string;
     teamId: string;
     reportedBy: string; // agent_id
     reportedByRole?: string;
     reportedAt: string; // ISO 8601
     summary: string;
     findings: TeamReportFinding[];
     context?: Record<string, unknown>;
   }
   ```
   実フィールドは `src-tauri/src/commands/team_state.rs` または `src-tauri/src/team_hub/protocol/tools/report.rs` の `TeamReportSnapshot` 定義から確認して 1:1 で写し取ること (取りこぼしがあると同種ズレが再発する)。
2. `TeamOrchestrationState` に `teamReports?: TeamReportSnapshot[]` を追加 (optional でレガシー state JSON との後方互換を担保)。
3. Canvas Dashboard / TeamDashboard での消費は別 issue で実施 (この issue は型同期に専念)。

### 案2: 型生成ツール (ts-rs / specta) を導入

- Rust 側 `#[derive(TS)]` 等で TS 型を自動 export し、再発を構造的に防ぐ。
- 工数大 (依存追加 + 既存型全量の再生成)。長期 refactor 候補だが Wave 1 では対象外、別 issue 化を提案。

### 副次対応 (本 issue 内で実施)

- TypeScript 側で `TeamOrchestrationState` の subkey が未定義 / undefined でも型安全にハンドリングできるよう、renderer 側の hook (`useTeamHistory` 等) で `teamReports ?? []` の defensive 読み出しを徹底。
