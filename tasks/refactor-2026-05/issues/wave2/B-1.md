TITLE: [bug] canvas: Background grid の color が CSS variable のため SVG attribute で評価されず固定色になる
LABELS: bug,canvas,javascript

## 概要 / Summary

Canvas モードの背景グリッドに渡している `color` 属性が `var(--canvas-grid, #1c1c20)` という CSS variable 形式の文字列のため、SVG attribute としては不正値扱いで評価されず、実質的に常に fallback / 固定色で描画される。テーマ切替で背景グリッドが追従しないほか、Issue #585 の「謎の縦線」「色が変」報告の根本原因の有力候補。

## 根本原因 / Root cause

- `src/renderer/src/components/canvas/Canvas.tsx:76, 431`
  ```ts
  const BACKGROUND_COLOR_VAR = 'var(--canvas-grid, #1c1c20)';
  // ...
  <Background color={BACKGROUND_COLOR_VAR} ... />
  ```
- `<Background>` (xyflow / @xyflow/react) は受け取った `color` を SVG `<circle fill={color}>` / `<path stroke={color}>` の attribute に直接書き込む。SVG attribute は CSS context ではないので `var(--canvas-grid, #1c1c20)` は CSS variable として展開されず、ブラウザは attribute 値を不正扱い (= fill が描画されない / 黒で描画される)。
- さらに `tokens.css` には `--canvas-grid` 自体が **未定義** で、現状はどのテーマでもブラウザ default になっている (Issue #585 の縦線・色見えに直結)。

## 再現手順 / Repro

1. vibe-editor を `claude-light` テーマで起動。
2. Canvas モードに切替 (Ctrl+Shift+M)。
3. 背景グリッドの dot / line 色を観察 (light テーマでも暗色ドット固定 / 縦線が出る等)。
4. DevTools で `<Background>` 内 SVG の `<circle fill="var(--canvas-grid, #1c1c20)">` を確認 → attribute として無視されている。

## 影響範囲 / Impact

- Canvas を使うすべての user に影響。
- テーマ切替で背景だけ追従しない (claude-light / glass / midnight 等で見栄えが崩れる)。
- Issue #585 の「縦線が出る」「色が変」の根本原因の最有力候補。
- 機能停止には至らないが、視覚上の theming 崩壊として常時表面化する HIGH bug。

## 修正方針 / Fix proposal

### 案 1: 直接 hex を渡す
`useTheme()` (Settings context 経由) でテーマ ID を読み、テーマごとに hex を分岐して `<Background color={...} />` に渡す。
- 短所: テーマ追加時に Canvas.tsx 側も更新する必要があり、4 層同期 (`themes.ts` / `tokens.css` / `monaco-setup.ts` / Canvas) に肥大化する。

### 案 2: `getComputedStyle` で variable 値を解決して渡す
mount 時に `getComputedStyle(document.documentElement).getPropertyValue('--canvas-grid')` を読んで渡す。
- 短所: テーマ切替時に再計算が必要 + xyflow の Background は color prop の変更で再描画が必要。

### 案 3 (推奨): `tokens.css` に `--canvas-grid` を定義し className 経由で SVG fill を上書き
1. `src/renderer/src/styles/tokens.css` の各 `[data-theme='...']` ブロックに `--canvas-grid` を追加:
   ```css
   :root, [data-theme='claude-dark'] { --canvas-grid: #2a2a2e; }
   [data-theme='claude-light']        { --canvas-grid: #d4d4d8; }
   [data-theme='dark']                { --canvas-grid: #1c1c20; }
   [data-theme='light']               { --canvas-grid: #d4d4d8; }
   [data-theme='midnight']            { --canvas-grid: #15151a; }
   [data-theme='glass']               { --canvas-grid: rgba(255,255,255,0.18); }
   ```
2. `src/renderer/src/styles/components/canvas.css` (または該当 CSS) に:
   ```css
   .react-flow__background-pattern circle,
   .react-flow__background-pattern path {
     fill: var(--canvas-grid, #1c1c20);
     stroke: var(--canvas-grid, #1c1c20);
   }
   ```
3. `Canvas.tsx` の `<Background>` から `color` prop を外す (xyflow が attribute で `color` を上書きしないようにする)。`BACKGROUND_COLOR_VAR` 定数も削除。

これにより SVG fill は CSS context で評価され、テーマ切替時の `data-theme` 属性更新だけでグリッド色が追従する。新規テーマを足すときも `tokens.css` に `--canvas-grid` を 1 行足すだけで済む (theme-customization skill の 4 層同期と整合)。
