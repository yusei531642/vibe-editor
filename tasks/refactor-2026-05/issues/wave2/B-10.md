TITLE: [bug] teamhub: inject() 中の user 入力混入を防ぐため set_injecting(true/false) を必ず呼ぶ
LABELS: bug,rust,backend

## 概要 / Summary

`team_hub::inject::inject_once` が bracketed-paste を流す間、`SessionHandle::set_injecting(true)` を呼んでいないため、ユーザーが端末に普通にキー入力すると bracketed paste の途中バイトに user 入力が混ざる。Windows ConPTY は 64B/15ms 制約のため、Linux より露出時間が長く再現性が高い。team_send で 3-5 KB の handoff を流すと数百 ms 〜 数秒間 user 入力が無効化されないまま inject が進む。

## 根本原因 / Root cause

- `src-tauri/src/team_hub/inject.rs:312-439` の `inject_once`:
  - bracketed-paste 開始 (`ESC[200~`) → 64B/15ms チャンクで本文 → `ESC[201~` → `\r` の流れ。
  - この間 `session.injecting` は false のまま。
- `src-tauri/src/pty/session.rs:65-69` で `injecting: AtomicBool` フィールドが定義されているが、セットする箇所がない。
- `src-tauri/src/commands/terminal.rs:104-138` で `terminal_write` 経路は `injecting == false` のときだけ書き込む設計になっているが、`set_injecting(true)` を inject 側が呼ばないので **常に false = ユーザー入力スルー**。
- 結果、bracketed paste の中ほどに user の `Enter` や任意キーが混入し、worker が壊れた prompt を受け取る。Issue #574 / #576 周辺で表面化したケースの遠因。

## 再現手順 / Repro

1. Canvas で leader-worker (recruit 完了後)。
2. leader から `team_send` で 3-5 KB の長文 handoff を流す。
3. 同時に worker terminal を click してユーザーが文字 (例: `h e l l o\n`) を打つ。
4. worker prompt の inject 結果に user 入力が紛れ込む / `Enter` で paste が早期確定する。

## 影響範囲 / Impact

- マルチエージェント機能 (vibe-team) を使う全ユーザー。
- 大きな handoff ほど露出時間が長く、worker prompt 破損が頻発。
- worker が誤った prompt で起動 → 期待しない作業 / 暴走 → 信頼性損失。

## 修正方針 / Fix proposal

### 推奨: RAII guard `InjectingGuard` で `set_injecting(true/false)` を必ずペアで呼ぶ

1. `src-tauri/src/pty/session.rs` に `InjectingGuard` を追加:
   ```rust
   pub struct InjectingGuard<'a> {
       session: &'a SessionHandle,
   }
   impl<'a> InjectingGuard<'a> {
       pub fn new(s: &'a SessionHandle) -> Self {
           s.injecting.store(true, Ordering::SeqCst);
           Self { session: s }
       }
   }
   impl<'a> Drop for InjectingGuard<'a> {
       fn drop(&mut self) {
           self.session.injecting.store(false, Ordering::SeqCst);
       }
   }
   impl SessionHandle {
       pub fn begin_injecting(&self) -> InjectingGuard<'_> { InjectingGuard::new(self) }
   }
   ```
2. `team_hub::inject::inject_once` 冒頭で:
   ```rust
   let _guard = session.begin_injecting();
   // 早期 return / panic / retry path すべてで guard が drop されて自動解放
   ```
3. `terminal_write` (`src-tauri/src/commands/terminal.rs:104-138`) で `if session.is_injecting() { return Err(...) }` ガードがあれば確認 (なければ追加)。
4. `inject_codex_prompt_to_pty` (B-11) も同 guard を必ず使う。

### 共通化 (Refactor opportunity, Tier D との接続)

`pty::session::SessionHandle::inject_text(text: &str)` のような共通 helper を抽出し、`team_hub::inject::inject_once` と `commands/terminal::inject_codex_prompt_to_pty` (B-11) の両方をその helper に置き換える。helper 内部で guard を必ず張り、bracketed paste / chunk size / spawn_blocking パターンを統一。これにより set_injecting の呼び忘れも防げる。
