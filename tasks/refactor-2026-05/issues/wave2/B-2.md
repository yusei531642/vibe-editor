TITLE: [bug] canvas: Team preset (#522) の Apply が teamId/agentId/setupTeamMcp 抜きで standalone agent を作る
LABELS: bug,canvas,javascript

## 概要 / Summary

`TeamPresetsPanel` の `handleApply` (#522 で導入されたユーザー定義 team preset を Canvas に展開する経路) が、本来 `BUILTIN_PRESETS` の適用と等価であるべきにも関わらず、`teamId` 発行 / `agentId` 発行 / `setupTeamMcp` 呼び出し / `placeBatchAwayFromNodes` をすべて抜かしている。結果として配置された agent カードはチームに属さない standalone agent となり、AgentNodeCard の `--append-system-prompt` も付かず、handoff / role 連携が一切働かない regression。

## 根本原因 / Root cause

- `src/renderer/src/components/canvas/TeamPresetsPanel.tsx:186-217` の `handleApply` は preset の `agents[]` をそのまま `addCards` に流し込むだけで:
  - `teamId = team-${randomUUID()}` の発行なし
  - 各 agent の `agentId` 発行なし
  - `setupTeamMcp` (`mcp_config::team_mcp::setup`) 呼び出しなし
  - 配置は素の grid 計算で `placeBatchAwayFromNodes` 経由でない (重なる)
- 一方、本来の組み込み preset 適用ロジックは `src/renderer/src/layouts/CanvasLayout.tsx:184-241` の `applyPreset` にあり、上記 4 点をすべて行っている。
- AgentNodeCard 側の sysPrompt 構築 (`src/renderer/src/components/canvas/cards/AgentNodeCard/CardFrame.tsx:278-312`) は:
  ```ts
  if (!payload.teamId || !payload.agentId || !teamMembers) return undefined;
  ```
  で `--append-system-prompt` 全体が undefined になる。つまり handleApply 経由の agent はロール強制や handoff protocol が当たらない。

## 再現手順 / Repro

1. Canvas モードを開き、TeamPresetsPanel から builtin 以外のユーザー定義 preset を Apply する。
2. 配置された agent カードを観察すると、複数カードが同位置にスタックしている。
3. AgentNodeCard を起動 (Recruit) → `--append-system-prompt` が付与されない (Claude 本体の system prompt のみで起動する)。
4. 別の agent から `team_send` を打っても handoff が成立しない (teamId 不一致)。

## 影響範囲 / Impact

- ユーザー定義 team preset (#522) を使う全ユーザーで、apply 後の team 機能が機能不全。
- teamId が無いため `setupTeamMcp` も走らず、`team_*` MCP tools が当該 agent から見えない (role 強制 / handoff / status report 全滅)。
- builtin preset と user preset で結果が大きく異なる「謎仕様」となり、preset 機能の信頼を損なう。

## 修正方針 / Fix proposal

### 推奨: 共通 helper `spawnTeam` を抽出して両経路で共有

1. `src/renderer/src/lib/canvas/spawn-team.ts` (新設) に共通関数を作る:
   ```ts
   export interface SpawnTeamInput {
     leaderRole: string;
     workers: { role: string; engine: 'claude' | 'codex'; ... }[];
     basePosition?: { x: number; y: number };
     existingNodes: Node<CardData>[];
     projectRoot: string;
     mcpAutoSetup: boolean;
   }
   export async function spawnTeam(input: SpawnTeamInput): Promise<{
     teamId: string;
     agents: { id: string; agentId: string; position: { x: number; y: number }; ... }[];
   }> {
     const teamId = `team-${crypto.randomUUID()}`;
     const positions = placeBatchAwayFromNodes(input.existingNodes, input.workers.length + 1, input.basePosition);
     if (input.mcpAutoSetup) {
       await invoke('app_setup_team_mcp', { projectRoot: input.projectRoot, teamId });
     }
     return {
       teamId,
       agents: [
         { id: `agent-${crypto.randomUUID()}`, agentId: `leader-${...}`, position: positions[0], ... },
         ...input.workers.map((w, i) => ({ ... position: positions[i + 1] ... })),
       ],
     };
   }
   ```
2. `CanvasLayout.tsx:applyPreset` と `TeamPresetsPanel.tsx:handleApply` を `spawnTeam` 呼び出しに統一。
3. `CanvasSidebar.tsx:handleResumeTeam` (B-3) も同 helper の variant (`spawnTeamFromHistory`) として相乗りする (B-3 と統合できる)。

### 短期 minimal fix
`handleApply` 内で `applyPreset` と同じ 4 つの処理 (teamId 発行 / setupTeamMcp / agentId 発行 / placeBatchAwayFromNodes) を直書きで追加する。後続 issue (B-3 含む) で共通化する。
