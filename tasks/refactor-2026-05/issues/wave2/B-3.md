TITLE: [bug] canvas: CanvasSidebar の handleResumeTeam が古いピッチ + placement 抜きで重複配置する
LABELS: bug,canvas,javascript

## 概要 / Summary

`CanvasSidebar.handleResumeTeam` (履歴から team を Canvas 上に再配置する経路) が hardcode された 520x360 ピッチで grid 配置しており、Issue #497 で確定した現行カードサイズ (NODE_W=760, NODE_H=460) とミスマッチしているため既存カードと完全に重なる。さらに `placeBatchAwayFromNodes` を経由しないため、ユーザーが既存配置で作業中の Canvas に既に置かれているカードと衝突する。`latestHandoff` を payload に同梱もしないので、resume 後の AgentNodeCard が空コンテキストで起動する。

## 根本原因 / Root cause

- `src/renderer/src/components/canvas/CanvasSidebar.tsx:142-195` の `handleResumeTeam`:
  ```ts
  const fallbackPosition = (i: number) => ({
    x: (i % 3) * 520,
    y: Math.floor(i / 3) * 360,
  });
  ```
  - 520x360 は #497 以前の旧カードサイズに合わせたピッチ。
  - 現行は `NODE_W=760`, `NODE_H=460` (canvas/constants 等参照) でカード自身の幅高より狭い grid → 確実に重なる。
- `placeBatchAwayFromNodes` を呼ばないため、既存 Canvas にカードが点在している場合は `(0,0), (520,0), (1040,0)` 付近にも上書きされる。
- B-2 と同様に `latestHandoff` / `unreadInboxCount` などの payload を同梱しないため、resume 後の agent カードは過去の handoff コンテキストを失う。

## 再現手順 / Repro

1. Canvas で複数 agent を散りばめた状態で作業し、team を実行 → 履歴に保存。
2. Canvas を一度 clear するか、別の team を上に展開して画面が埋まっている状態にする。
3. CanvasSidebar の team 履歴から `Resume` を押す。
4. 配置された team agent カードが既存カードと完全に重なって見えなくなる、もしくは元 team の `latestHandoff` が出ないので worker からの返答コンテキストが空。

## 影響範囲 / Impact

- 履歴経由の team 復元を使う全ユーザーで配置事故が発生。
- 既存カードに上書きされるため、ユーザーは「team が消えた」と誤認しやすい。
- handoff コンテキスト欠落で resume 後の Recruit 動作が連続性を持たない (再起動と同じ状態)。

## 修正方針 / Fix proposal

### 推奨: B-2 と同じ `spawnTeam` helper を共有 (variant `spawnTeamFromHistory`)

1. B-2 の Fix で抽出する `spawnTeam` を `spawnTeamFromHistory(historyEntry, existingNodes)` の形でも呼べるよう拡張する:
   ```ts
   export async function spawnTeamFromHistory(
     entry: TeamHistoryEntry,
     existingNodes: Node<CardData>[],
   ): Promise<{ nodes: Node<CardData>[] }> {
     const positions = placeBatchAwayFromNodes(existingNodes, entry.agents.length);
     return {
       nodes: entry.agents.map((a, i) => ({
         id: `agent-${crypto.randomUUID()}`,
         type: 'agent',
         position: positions[i],
         data: {
           type: 'agent',
           payload: {
             teamId: entry.teamId,                      // 履歴の teamId を保持
             agentId: a.agentId,
             role: a.role,
             engine: a.engine,
             resumeSessionId: a.resumeSessionId,        // claude --resume 用
             latestHandoff: entry.latestHandoffByAgent?.[a.agentId], // 同梱
             unreadInboxCount: 0,
           },
         },
       })),
     };
   }
   ```
2. `handleResumeTeam` 内の grid fallback (520/360) を削除し、`spawnTeamFromHistory` の戻り値で `addCards` する。
3. `placeBatchAwayFromNodes` 起点 position (基準点) は現在の viewport center を使う (xyflow `useReactFlow().getViewport()`)。

### 短期 minimal fix (B-2 統合まで延期する場合)

- 520/360 を `NODE_W + NODE_GAP_X` / `NODE_H + NODE_GAP_Y` (定数化) に置換。
- `placeBatchAwayFromNodes(existingNodes, agents.length)` を経由する。
- `latestHandoff` / `unreadInboxCount` を payload に同梱。
