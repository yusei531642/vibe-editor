TITLE: [bug] canvas: Canvas の Ctrl+Shift+K/I/N が IDE モード中も発火し DevTools / IDE 操作を奪う
LABELS: bug,canvas,ui,javascript

## 概要 / Summary

Canvas 用の useKeybinding (Ctrl+Shift+K の QuickNav, Ctrl+Shift+N の新規 agent カード追加, Ctrl+Shift+I の Inspector) が IDE モード中も capture phase で listen しており、IDE 中に押下すると Canvas モードの state が裏で書き換わるほか、`Ctrl+Shift+I` は Chromium 標準の DevTools ショートカットを奪ってしまう。`<CanvasLayout />` 自体は IDE モードでも常時 mount されているため、`<Canvas />` の useKeybinding が常に有効になっているのが原因。

## 根本原因 / Root cause

- `<CanvasLayout />` は `App.tsx` 内で常時 mount され、`<Canvas />` も常に存在する (`viewMode` 切替で表示 / 非表示が変わるだけ)。
  - 参考: `src/renderer/src/main.tsx:185-196`, `src/renderer/src/layouts/CanvasLayout.tsx:69-91`
- `src/renderer/src/components/canvas/Canvas.tsx:325-328` 周辺で:
  ```ts
  useKeybinding(KEYS.quickNav, () => setShowQuickNav(true));
  useKeybinding(KEYS.canvasNewAgent, () => addAgent(...));
  useKeybinding(KEYS.canvasInspector, () => setShowInspector(true));
  ```
  などが `enabled` 引数なしで登録されており、`useKeybinding` 内部は capture phase で `window.addEventListener('keydown', ..., true)` する。
- IDE モード中も発火するため:
  - `Ctrl+Shift+K`: QuickNav overlay state がリーク (見えないが store に乗る)。
  - `Ctrl+Shift+N`: invisible な agent カードが Canvas store に積まれる (Canvas に切り戻すと突然増えている)。
  - `Ctrl+Shift+I`: Chromium DevTools の Ctrl+Shift+I を奪い、開発時のデバッグを阻害する (本番でも DevTools 開けるべきではないが、開発時の体験を直接損なう)。

## 再現手順 / Repro

1. 新規 vibe-editor 起動 (IDE モードで開始する想定)。
2. 任意のファイルを開いた状態で `Ctrl+Shift+N` を連打する。
3. `Ctrl+Shift+M` で Canvas モードに切替 → 大量の空 agent カードが既に積まれている。
4. IDE モードで `Ctrl+Shift+I` を押下 → DevTools が開かず Canvas Inspector overlay state が裏で立ち上がる。
5. IDE モードで `Ctrl+Shift+K` → QuickNav が表示されない (Canvas にいない) が store の `showQuickNav=true` になる。

## 影響範囲 / Impact

- 全ユーザーの IDE モード体験を損なう (頻用ショートカットが裏で副作用を起こす)。
- 開発者が DevTools を開けない / 開きにくい (Ctrl+Shift+I)。
- Canvas store が予期せず汚れ、IDE → Canvas 切替で大量の空ノードを発見する事故。
- vibeeditor skill / canvas-nodecard-pattern skill のキー設計前提を逸脱。

## 修正方針 / Fix proposal

### 推奨: `useKeybinding` の `enabled` 引数を活用し `viewMode === 'canvas'` のときだけ listen

1. `src/renderer/src/stores/ui.ts` の `useUiStore` を `Canvas.tsx` で読む:
   ```ts
   import { useUiStore } from '@/stores/ui';
   const viewMode = useUiStore(s => s.viewMode);
   const isCanvas = viewMode === 'canvas';
   ```
2. 各 useKeybinding 呼び出しに `enabled: isCanvas` を渡す:
   ```ts
   useKeybinding(KEYS.quickNav, () => setShowQuickNav(true), { enabled: isCanvas });
   useKeybinding(KEYS.canvasNewAgent, () => addAgent(...), { enabled: isCanvas });
   useKeybinding(KEYS.canvasInspector, () => setShowInspector(true), { enabled: isCanvas });
   ```
3. `useKeybinding` 自体に `enabled: false` のときは `addEventListener` を呼ばない (or no-op で early return) ガードを足す (既にあれば再確認)。
4. 同様のパターンが Canvas 専用ショートカット (Ctrl+Shift+E, etc.) で他にあれば一斉に修正。

### 補強: `viewMode` を `useKeybinding` レベルで guard する `useCanvasKeybinding(...)` ヘルパを新設し、Canvas モードでないと発火しない wrapper として一元化する案もある (ただし overhead に対し効果は限定的)。
