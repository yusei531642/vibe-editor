TITLE: [bug] canvas: 狭い画面で HUD ボタンが flex-shrink+word-break で日本語縦書き化
LABELS: bug,canvas,ui,a11y

## 概要 / Summary

Canvas モードの StageHud (画面下部中央のフローティング HUD) のボタンに `white-space: nowrap` が無く、HUD 全体が画面幅に収まらない狭幅環境では各ボタンが `flex-shrink: 1` で潰され、内側 `<span>` の日本語が word-break で 1 文字ずつ折り返されて視覚的に縦書きになる。Issue #586 の「HUD 表示が変」報告の主原因。

## 根本原因 / Root cause

- `src/renderer/src/components/canvas/StageHud.tsx:301-313` の HUD ボタン群は flex container 内に配置。
- `src/renderer/src/styles/components/canvas.css:1109-1121` 付近の `.tc__hud` 関連スタイル:
  - HUD は `position: absolute; bottom: 22px; left: 50%; transform: translateX(-50%)`。
  - `max-width` 指定なし、`flex-wrap: nowrap` も指定なし。
  - 内側 `<button>` には `flex-shrink` を指定していない (= デフォルト `flex-shrink: 1`)。
  - `<button>` 内の `<span>` ラベルにも `white-space: nowrap` 指定なし → 日本語は CJK の特性で 1 文字単位で折り返される。
- 画面幅が HUD コンテンツ合計幅未満になると、flex shrink で各ボタンが潰れ、内部 `<span>` が縦書きに見える。

## 再現手順 / Repro

1. Canvas モードを開く。
2. ウィンドウ幅を 900px 以下まで縮める (もしくは density=comfortable + 多 button preset)。
3. 画面下部の HUD ボタン (例: 「チーム解散」「Recruit 警告」等) のラベルが縦に積まれて日本語縦書きに見える。
4. DevTools で各 button を inspect → `width` が極端に狭く (20-30px)、`<span>` の文字が wrap しているのが確認できる。

## 影響範囲 / Impact

- ノートPC / 13 inch 以下のサブモニター / DPI 高設定でデフォルト density のユーザーで頻発。
- 操作不能になるわけではないが、ラベルが読めず UX が大幅劣化。a11y / 視認性で問題。
- Issue #586 の「HUD 表示変」報告の根本原因。

## 修正方針 / Fix proposal

### 推奨: HUD ボタンに `flex-shrink: 0` + `white-space: nowrap` を強制し、はみ出し時は HUD 全体を横スクロール可能にする

1. `src/renderer/src/styles/components/canvas.css` の HUD ブロックに:
   ```css
   .tc__hud {
     max-width: calc(100vw - 32px);
     overflow-x: auto;        /* はみ出し時は横スクロール */
     scrollbar-width: thin;
   }
   .tc__hud button {
     flex-shrink: 0;           /* 縮まない */
     white-space: nowrap;      /* ボタン全体で折り返さない */
   }
   .tc__hud button > span,
   .tc__hud button > * {
     white-space: nowrap;      /* 子の span でも折り返さない */
   }
   ```
2. 補助: density=compact 時には一部のボタンを `display: none` または icon-only に切り替えるレスポンシブ対応 (future enhancement、本 issue では必須ではない)。
3. B-6 (HUD 複数 team 集約) と並行する場合は HUD レイアウト全体のリファクタを 1 PR でまとめても良い。

### 代替: HUD を垂直方向にレイアウト切替 (狭幅時 column flex)
- 縦に積むと bottom 位置から重なるレイアウトに変わり今度は Canvas 視認領域を圧迫するため、推奨案より劣る。
