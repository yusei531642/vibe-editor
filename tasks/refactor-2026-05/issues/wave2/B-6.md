TITLE: [bug] canvas: HUD / TeamDashboard が複数 team の集約に対応していない (dual preset で片方の dead count が消える)
LABELS: bug,canvas,javascript

## 概要 / Summary

`StageHud` の `aggregatedTeamId` が「最初に見つけた leader、無ければ最初の team」を 1 つだけ返す実装になっているため、`BUILTIN_PRESETS` の `dual-claude-claude` / `dual-claude-codex` / `dual-codex-codex` / `dual-codex-claude` といった「2 組織同時起動」preset を展開すると、片方の team が HUD / TeamDashboard の health 集計から完全に脱落する。dead count や recruit 警告が片方分しか出ないため、ユーザーは静かに片肺のまま気付かず作業を続けてしまう。

## 根本原因 / Root cause

- `src/renderer/src/components/canvas/StageHud.tsx:187-227` の `aggregatedTeamId`:
  ```ts
  const teams = collectTeams(nodes); // [{ teamId, hasLeader }, ...]
  const leaderTeam = teams.find(t => t.hasLeader);
  const aggregatedTeamId = leaderTeam?.teamId ?? teams[0]?.teamId ?? null;
  ```
  返すのは 1 つの teamId のみ。
- `useTeamHealth(aggregatedTeamId)` は単一 teamId を引数に取り、HUD / TeamDashboard の `deadCount` / `staleCount` 等を 1 team 分しか集計しない。
- `BUILTIN_PRESETS` (`src/renderer/src/lib/workspace-presets.ts:86-169`) には dual preset が 4 種定義されており、それぞれが 2 つの独立した teamId を Canvas 上に展開する設計。
- 結果として 2 つ目以降の team の死活情報・stale 警告は HUD / dashboard 上に一切出ない。

## 再現手順 / Repro

1. Canvas モードで `dual-claude-claude` preset を Apply。
2. 2 つの team (4-6 agent) が並んで配置される。
3. 片方の team の leader を kill (端末を閉じる、process kill 等) する。
4. HUD の dead count バッジを観察 → kill された team が leader team でなければ dead count が `0` のまま増えない。

## 影響範囲 / Impact

- dual preset を使うユーザー (preset 機能の主な訴求点の 1 つ) で health 集計が 1/2 しか表示されず、片肺障害を見逃す。
- 「Canvas を見ながら回している」前提が崩れ、信頼性が大きく低下。
- multi-team 構成は今後の拡張 (3 team 以上) でも同じ問題を引き起こす。

## 修正方針 / Fix proposal

### 推奨: `useTeamHealth` を multi-team 対応にし、HUD / Dashboard を集約

1. `src/renderer/src/lib/hooks/use-team-health.ts` を多 team 対応に変更:
   ```ts
   export interface MultiTeamHealth {
     totalDead: number;
     totalStale: number;
     perTeam: Map<string, { dead: number; stale: number; ... }>;
   }
   export function useTeamHealthMulti(teamIds: string[]): MultiTeamHealth { ... }
   ```
   既存 `useTeamHealth(teamId)` は thin wrapper として残す (後方互換)。
2. `StageHud.tsx` で `aggregatedTeamId` を `aggregatedTeamIds: string[]` に置き換え:
   ```ts
   const teams = collectTeams(nodes);
   const aggregatedTeamIds = teams.map(t => t.teamId);
   const health = useTeamHealthMulti(aggregatedTeamIds);
   ```
3. HUD バッジ (`deadCount`, `staleCount`) は `health.totalDead` / `totalStale` を表示。
4. TeamDashboard では team ごとに分割表示 (`health.perTeam`) し、leader 不在 team も表示できるようにする。
5. `dashboardTeamId` (Inspector を開く際の対象 team 切替) は user が任意の team を選択できる UI に拡張するか、active team list を tabbed で表示。

### 短期 minimal fix
- `aggregatedTeamId` をやめて `useMemo(() => teams.map(t => t.teamId), [...])` で配列化、HUD では `teamIds.reduce((acc, id) => acc + teamHealth(id).dead, 0)` の単純合計だけ実装。Inspector は当面 leaderTeam にフォールバック。
