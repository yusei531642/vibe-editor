TITLE: [bug] canvas: Pane 右クリックメニューが mousedown の伝播競合で開かない / 閉じない
LABELS: bug,canvas,ui,javascript

## 概要 / Summary

Canvas の Pane (空白部分) を右クリックした際、コンテキストメニューが意図通り開かない / 開いた直後に閉じてしまう / 左クリックで閉じれない、という状態が起きる。原因は `handlePaneContextMenu` が `e.preventDefault()` だけ呼んで `e.stopPropagation()` を呼ばないため、ContextMenu の outside-click 検知 (`mousedown` listener) が「開いたきっかけの右クリック自体」を outside-click として検出してしまう。Issue #593 の右クリックメニュー閉じれない症状の根本原因。

## 根本原因 / Root cause

- `src/renderer/src/components/canvas/Canvas.tsx:279-291` の `handlePaneContextMenu`:
  ```ts
  const handlePaneContextMenu = (e: React.MouseEvent) => {
    e.preventDefault();
    setContextMenu({ x: e.clientX, y: e.clientY, target: 'pane' });
  };
  ```
  `e.stopPropagation()` を呼んでいない。一方 `handleNodeContextMenu` (line 256 周辺) は両方呼んでいる (= node 上の右クリックは正常動作)。
- `src/renderer/src/components/ContextMenu.tsx:69-112` の outside-click 検知:
  ```ts
  useEffect(() => {
    const onMouseDown = (e: MouseEvent) => {
      if (!ref.current?.contains(e.target as Node)) onClose();
    };
    document.addEventListener('mousedown', onMouseDown);
    ...
  }, []);
  ```
  - `mousedown` で外クリックを検知している。React の synthetic event は capture phase 完了後に bubble するため、`mousedown` 自体が ContextMenu 表示直後 (1 frame 以内) に listener にフックされる。
  - React Flow の Pane は contextmenu イベントを `bubble` させるため、document への `mousedown` 登録時点で「同じ右クリックの mousedown」が「ContextMenu 外の mousedown」として誤検知される。
- 結果: ContextMenu が mount された frame で mousedown listener が走り、即 `onClose()` が呼ばれて閉じる。あるいは、初回 click では state は立つが mount 後 1 tick で閉じてユーザーには「開かない」と見える。

## 再現手順 / Repro

1. Canvas を開き、Pane (空白) を右クリック。
2. メニューが一瞬開いて即閉じる (フリッカー)、もしくは全く開かない。
3. 開いた状態で別の場所を左クリックしてもまれに閉じない (fcous lag)。
4. node 上 (AgentNodeCard 等) で右クリックすると正常動作する → handlePaneContextMenu のみの問題と切り分け可能。

## 影響範囲 / Impact

- Canvas Pane の右クリックメニュー全機能が事実上使えない / 不安定 (#593 の根本)。
- 「カードを Pane に追加」「viewport reset」等のコンテキスト操作が阻害される。
- ユーザー体験のリグレッション源。

## 修正方針 / Fix proposal

### 推奨: 両方修正 (defense-in-depth)

1. `handlePaneContextMenu` に `e.stopPropagation()` を追加:
   ```ts
   const handlePaneContextMenu = (e: React.MouseEvent) => {
     e.preventDefault();
     e.stopPropagation();
     setContextMenu({ x: e.clientX, y: e.clientY, target: 'pane' });
   };
   ```
2. ContextMenu の outside-click 検知を `mousedown` から `click` (mouseup 完了後) に変更:
   ```ts
   useEffect(() => {
     const onClick = (e: MouseEvent) => {
       if (!ref.current?.contains(e.target as Node)) onClose();
     };
     document.addEventListener('click', onClick);
     return () => document.removeEventListener('click', onClick);
   }, []);
   ```
   または `useEffect` の中で `setTimeout(() => { addEventListener('mousedown', ...) }, 0)` で「現在の event loop tick が終わってから listen 開始」にする。
3. (任意) `pointerdown` を使う形に統一する案もあるが、互換性的に `click` が無難。

### 短期 minimal
1 のみ適用するだけでも症状は概ね解消するが、他の経路 (右クリックジェスチャ) で再発しうるので 2 を併せて対応する。
