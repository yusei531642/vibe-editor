TITLE: [bug] filetree: primaryRoot を workspace から外す UI が存在しない
LABELS: bug,ui,javascript

## 概要 / Summary

`FileTreePanel` の roots 一覧で、primary (現在のプロジェクト) ルートには削除ボタンが描画されない条件分岐 (`{!isPrimary && (<button>)}`) が入っているため、ユーザーが「現在開いているプロジェクトを workspace から外す」「別のプロジェクトを primary に切り替える」操作ができない。Issue #591 の「フォルダ外せない」報告の根本。

## 根本原因 / Root cause

- `src/renderer/src/components/FileTreePanel.tsx:303-334`:
  ```tsx
  {roots.map((root, i) => {
    const isPrimary = i === 0;
    return (
      <div className="filetree__root">
        <span>{root}</span>
        {!isPrimary && (
          <button className="filetree__root-remove" onClick={() => removeRoot(root)}>×</button>
        )}
      </div>
    );
  })}
  ```
  `roots` 配列は `[primaryRoot, ...extraRoots]`。`isPrimary` は常に index 0 (primary) のみを意味し、削除ボタンが描画されない。
- 結果として「最後に開いたプロジェクトを workspace から完全に外す」「別のサブフォルダを primary に格上げする」UI が存在しない。CLI からは `~/.vibe-editor/settings.json` を手動編集しなければならない。

## 再現手順 / Repro

1. vibe-editor を起動し、何らかのプロジェクトを開く。
2. FileTreePanel に primary root (現在のプロジェクト) と、もし extra roots があれば併記される。
3. primary root の右側を hover しても × ボタンが現れない。
4. extra root には × ボタンがある (差異が確認できる)。
5. 「別のプロジェクトに切り替えてから古い primary を外したい」シナリオが完結できない。

## 影響範囲 / Impact

- workspace 整理ができないため、ユーザーは「使い終わったプロジェクトを永久に primary として保持」してしまう。
- 「プロジェクト未選択」状態に戻す UX が無いため、新規プロジェクト体験 (OnboardingWizard 復帰) も成立しない。
- Issue #591 の根本原因。

## 修正方針 / Fix proposal

### 推奨: primary root にも削除ボタンを描画し、押下時の挙動を 2 段階で扱う

1. `FileTreePanel.tsx` の primary 判定をなくして全 root に × ボタンを描画:
   ```tsx
   {roots.map((root, i) => {
     const isPrimary = i === 0;
     return (
       <div className="filetree__root">
         <span>{root}</span>
         <button
           className="filetree__root-remove"
           onClick={() => handleRemoveRoot(root, isPrimary)}
           title={isPrimary ? 'このプロジェクトを workspace から外す' : 'extra root を外す'}
         >×</button>
       </div>
     );
   })}
   ```
2. `handleRemoveRoot(root, isPrimary)` の挙動:
   - `isPrimary === false` (extra root): 既存の `removeRoot(root)` を呼ぶ。
   - `isPrimary === true`:
     - 他に extra root がある → 最初の extra root を `pushRecent` で primary に昇格し、当該 root を外す。
     - extra root が無い → `lastOpenedRoot = ''` にして「プロジェクト未選択」状態に戻す (OnboardingWizard を再表示するか、空状態の placeholder を出す)。
3. confirm dialog (「現在のプロジェクトを閉じます。よろしいですか？」) を primary 削除時のみ表示。

### 短期 minimal fix
primary root にも × ボタンを描画し、押下時には `lastOpenedRoot = ''` にしてプロジェクト未選択へ。confirm dialog だけ追加。
