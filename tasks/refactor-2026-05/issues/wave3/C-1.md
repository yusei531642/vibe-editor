TITLE: [bug] canvas: clear() 後 xyflow の viewport が store と desync する
LABELS: bug,canvas,javascript

## 概要 / Summary

Canvas モードで `clear()` を実行すると `useCanvasStore` 側の viewport は `{x:0,y:0,zoom:1}` にリセットされるが、xyflow 内部の viewport state (`React Flow` の uncontrolled reactive state) は最後にユーザーが pan / zoom した値のまま残るため、store と表示が desync する。リセットしたつもりが画面はずれたままになり、UX が分かりにくい。

## 根本原因 / Root cause

- `src/renderer/src/components/canvas/Canvas.tsx:321, 402`: `defaultViewport` を `useMemo([], [])` (空依存) で生成し、`<ReactFlow defaultViewport={defaultViewport} />` に渡している。
- xyflow の `defaultViewport` は **uncontrolled API** で、コンポーネントの mount 時に 1 度だけ参照される。以降は xyflow 内部 state が真であり、外から `defaultViewport` を変えても反映されない。
- `src/renderer/src/stores/canvas.ts:264` の `clear()` は store 側の viewport を `{x:0,y:0,zoom:1}` に書き戻すが、xyflow 内部に対して `setViewport` を発行しないので、表示は前の pan/zoom のまま固定される。

## 再現手順 / Repro

1. Canvas モードを開き、適当に pan / zoom してから複数のカードを置く。
2. Canvas HUD の `Clear` ボタン (`canvas.clearConfirm` を経由) で全クリアを実行。
3. 全カードは消えるが、画面の origin は元 (0,0) ではなく直前 pan/zoom 状態のまま残る。store devtools (zustand) で確認すると `viewport = {0,0,1}` だが xyflow は別位置を表示している。

## 影響範囲 / Impact

- Canvas を頻繁に使うユーザー全員 (clear 後に「画面が真っ黒で何もない」状態になる)
- preset Apply 直後に `placeBatchAwayFromNodes` で配置したカードが画面外に並び、ユーザーが見つけられない事故につながる
- severity: MEDIUM (機能不全ではないが UX バグとして高頻度に表面化)

## 修正方針 / Fix proposal

### 案1 (推奨): clear() を `useReactFlow().setViewport(...)` と組で発行する

1. `Canvas.tsx` 側で `const { setViewport } = useReactFlow()` を取得し、`clear()` を呼ぶラッパ `clearAndReset` を用意する:
   ```ts
   const clearAndReset = useCallback(() => {
     useCanvasStore.getState().clear();
     setViewport({ x: 0, y: 0, zoom: 1 }, { duration: 0 });
   }, [setViewport]);
   ```
2. HUD / コマンドパレットなどから直接 `useCanvasStore.getState().clear()` を呼んでいる箇所を `clearAndReset()` 呼び出しに統一。
3. store の `clear()` 自体には xyflow 依存を持たせず、UI 層 (Canvas.tsx) でのみ `setViewport` を発行する責務分離を維持。

### 案2: store に viewport reset signal を持たせ、Canvas.tsx で subscribe

`useCanvasStore` に `viewportResetTick: number` のような signal を導入し、`clear()` でインクリメント。Canvas.tsx の `useEffect([viewportResetTick])` で `setViewport` を呼ぶ。store 単体で viewport reset を発火できるが、indirection が増える。
