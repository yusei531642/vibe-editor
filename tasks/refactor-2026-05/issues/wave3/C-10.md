TITLE: [security] team_status: 頻度制限と長さ上限・control char 除去 (autoStale 偽装緩和)
LABELS: security,rust,backend

## 概要 / Summary

TeamHub の `team_status` MCP tool は permission check 無し / 頻度制限 無し / 長さ上限 無しで `current_status` (任意文字列) を `MemberDiagnostics` に書き込む。攻撃的な (または bug を含む) worker が高頻度で `team_status` を呼ぶことで `last_status_at` を常に新鮮に保ち、`autoStale` 検知 (Issue #574 系) を永久に false にできるほか、長文や制御文字を log に流して観測ノイズを生み出せる。

## 根本原因 / Root cause

- `src-tauri/src/team_hub/protocol/tools/status.rs:25-51`
- `team_status` tool の handler が:
  - permission check 無し (どの role からでも呼べる)
  - rate limit 無し (1 worker が 1 秒 1000 回連打可能)
  - `current_status` 文字列の長さ・charset チェック無し
  - `last_status_at` を unconditionally に現在時刻で更新するため、autoStale 判定 (Hub 側で「N 分間 status 来ない member は stale」) を簡単に bypass できる

## 再現手順 / Repro

1. team を recruit し、worker A から MCP `team_status({ current_status: "ok" })` を 1 秒間に 100 回 loop 投げる。
2. Hub の `MemberDiagnostics.last_status_at` は更新され続け、autoStale が永久に立たない。
3. `team_status({ current_status: "<\x1b[2J制御文字を含む長文>" })` のような payload も diagnostic に書ける → renderer 側で `team_diagnostics_read` を呼んで描画した際にエスケープシーケンスが流れる。

## 影響範囲 / Impact

- 悪意ある or バグった worker が autoStale 偽装で「死んだふり耐性」を得る
- `current_status` 経路で log injection / renderer escape (control chars)
- severity: MEDIUM (security / hardening)

## 修正方針 / Fix proposal

### 案1 (推奨): rate limit + length cap + control char strip

1. Hub state に `agent_id -> Instant` の `last_status_call_at` Map を持ち、3 秒以内の再呼び出しは silently 無視 (or 軽い debounce error)。
2. `current_status` を `sanitize_for_paste`-like (ESC/BEL/NUL 除去) + 長さ 256 byte 上限で trim。超過時は trim + `... (truncated)`。
3. `team_status` 内で `assert_member_role(ctx)` 等で「team の member であること」を最低限確認 (現状は token があれば誰でも呼べる)。

具体的には `src-tauri/src/team_hub/protocol/tools/status.rs` の冒頭に以下を追加:

```rust
const MAX_STATUS_LEN: usize = 256;
const MIN_STATUS_INTERVAL: Duration = Duration::from_secs(3);

if let Some(last) = state.last_status_at.get(&ctx.agent_id) {
    if last.elapsed() < MIN_STATUS_INTERVAL {
        return Ok(json!({ "ok": false, "reason": "rate_limited" }));
    }
}
state.last_status_at.insert(ctx.agent_id.clone(), Instant::now());

let mut s = sanitize_for_paste(&current_status);
if s.len() > MAX_STATUS_LEN {
    s.truncate(MAX_STATUS_LEN);
    s.push_str(" ... (truncated)");
}
```

### 案2: trust score model

worker ごとに trust score を持ち、頻度違反で score を下げる。score < threshold で `team_status` を拒否。複雑だが将来の DoS 対策の足場になる。
