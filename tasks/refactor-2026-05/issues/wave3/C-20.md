TITLE: [refactor] settings/role-profiles: .bak をタイムスタンプ + 世代回転に変更
LABELS: refactor,rust,backend,persistence,settings

## 概要 / Summary

`settings.json` / `role-profiles.json` の atomic write 経路は退避用 `.bak` ファイルを「最後の 1 世代」しか持たないため、連続して破損保存が走った場合に **健全な原本がたった 1 ステップで消える**。タイムスタンプ付き `.bak` で 5 世代程度を rotate する設計に変更し、ユーザーがいつでも復旧可能にする。

## 根本原因 / Root cause

- `src-tauri/src/commands/settings.rs:243-247`
- `src-tauri/src/commands/role_profiles.rs:28-32`
- `atomic_write` 系の helper が `*.bak` を 1 ファイルだけ保持する設計。
- 例: 健全な v1 → 破損保存 v2 → `.bak` = v1 / 本体 = v2 (破損)。次の保存で `.bak` = v2 (破損) / 本体 = v3 (破損) になり、原本 v1 が永久に失われる。
- 連続破損 (process kill / disk error / 不正な migration バグ) 時に復旧の最後の砦が消える。

## 再現手順 / Repro

1. 健全な settings.json (v1) でビルドして起動。
2. settings 保存ロジックに細工してわざと破損 JSON を書く (もしくは migration バグで invalid JSON を吐かせる)。
3. 1 回目の保存で `.bak` = v1 (健全) / `settings.json` = 破損。
4. 2 回目の保存で `.bak` = 破損版 / `settings.json` = また破損。
5. v1 (健全) がもう存在しない → 起動できなくなる / 設定が完全リセットされる。

## 影響範囲 / Impact

- 起動時 / settings 保存中の crash や migration バグに遭遇した user 全員
- `role-profiles.json` の dynamic role 永続化も同パスを通るため、role 設定の連鎖破損リスク
- severity: MEDIUM (refactor、頻度は低いが起きた時の損害大)

## 修正方針 / Fix proposal

### 案1 (推奨): タイムスタンプ + 5 世代回転

1. `atomic_write` の bak 退避を以下に変更:
   ```
   settings.json.2026-05-09T12-00-00.bak
   settings.json.2026-05-09T11-30-00.bak
   ...
   ```
   ファイル名にタイムスタンプを入れ、最古を rotate で削除 (5 世代まで保持)。

2. 共通 helper を `src-tauri/src/util/atomic_write.rs` に追加:
   ```rust
   pub fn rotate_backups(target: &Path, max_generations: usize) -> Result<()> {
       // 1. 既存の "<target>.<ts>.bak" を列挙して mtime 降順 sort
       // 2. max_generations を超える分を削除
       // 3. 現行 file を新しい timestamp 付き .bak に move
       Ok(())
   }
   ```

3. `settings_save` / `role_profiles_save` / `team_history_save` / `team_presets_save` すべてで使い回す (既存 SAVE_LOCK 4 重定義の DRY 解消とも合流)。

### 案2: git-style snapshot

`.vibe-editor/.history/` 配下に `<target>.<sha>.json` で保存し、index file で世代管理。実装が重いが復旧 UI を生やせる。Wave 3 では案1 で十分。

### 補強: 復旧 UI

設定モーダルに「最新の `.bak` から復元」ボタンを置く。タイムスタンプ付きなら一覧から選択可能 → power user 向けに有用。
