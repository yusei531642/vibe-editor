TITLE: [perf] canvas: useCanvasTeamRestore が node 追加のたびに O(N) チーム集計を再実行
LABELS: performance,canvas,javascript

## 概要 / Summary

`useCanvasTeamRestore` の effect が `nodes` を依存配列に持つため、Canvas にカードを 1 枚追加するたびに **全 nodes を走査して `byTeam` Map を再構築**する。実際に新規 IPC を発行する処理は `restoredTeamsRef` で de-dup されているため通信は 2 回起きないが、毎回の Map 再構築コストが node 数 × node 追加回数で蓄積する。

## 根本原因 / Root cause

- `src/renderer/src/lib/hooks/use-canvas-team-restore.ts:39-86`
- effect の deps が `[projectRoot, nodes, mcpAutoSetup]`。`nodes` 配列は addCard / drag / position 更新のたびに新しい参照になるため、effect は毎回再走する。
- effect 本体で `byTeam: Map<string, AgentNodeData[]>` を毎回ゼロから組み直しており O(N)。
- `restoredTeamsRef` で「既に restore 済みの teamId」を skip しているので IPC 重複は無いが、`byTeam` 構築自体は毎回走る。

## 再現手順 / Repro

1. Canvas に preset を 1 つ apply する (5-10 ノード)。
2. 続けて手動で agent を 1 枚ずつ追加していく。
3. React DevTools Profiler で `useCanvasTeamRestore` を観察すると、addCard 1 回ごとに effect が走り Map 再構築が発生していることが見える。
4. 100 枚規模になると visible に lag が出る。

## 影響範囲 / Impact

- Canvas で大量カードを扱うパワーユーザー
- カード追加時の体感応答性低下
- severity: MEDIUM (perf、機能不全ではない)

## 修正方針 / Fix proposal

### 案1 (推奨): `byTeam` 計算を `useMemo([nodes])` で囲み、effect 本体は差分判定だけにする

```ts
const byTeam = useMemo(() => {
  const map = new Map<string, AgentNodeData[]>();
  for (const n of nodes) {
    if (n.type !== 'agent') continue;
    const teamId = n.data?.payload?.teamId;
    if (!teamId) continue;
    const arr = map.get(teamId) ?? [];
    arr.push(n.data);
    map.set(teamId, arr);
  }
  return map;
}, [nodes]);

useEffect(() => {
  for (const [teamId, members] of byTeam) {
    if (restoredTeamsRef.current.has(teamId)) continue;
    // ... call setupTeamMcp ...
    restoredTeamsRef.current.add(teamId);
  }
}, [byTeam, projectRoot, mcpAutoSetup]);
```

これで `byTeam` 自体は `nodes` の変化で再計算されるが、useMemo の return がメモ化される。effect は `byTeam` 参照が変わったときだけ走る (内容が同じなら React は再呼び出しを skip しないが、本体は新規 teamId が無ければ noop)。

### 案2: canvas store に `teamsById` 派生 selector を生やす

zustand の `subscribeWithSelector` で `nodes` から `teamsById: Map<...>` を派生させ、`useCanvasTeamRestore` はそれだけを subscribe する。drag による position 変化では参照が変わらないので effect 自体が走らない。
