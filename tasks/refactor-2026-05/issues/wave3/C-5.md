TITLE: [bug] canvas: subscribeOnVisible が短時間の focus パカパカで誤発火し recruit warning toast が早期 flush する
LABELS: bug,canvas,javascript

## 概要 / Summary

`subscribeOnVisible` は window のフォーカス / 可視状態変化を購読するモジュール singleton だが、短時間で active / inactive を繰り返した場合に flush が複数回 race で発火する。結果、Issue #578 で導入した「Canvas が非表示の間に発生した recruit warning を可視化時にまとめて toast で見せる」機能が、ユーザーが見ていない一瞬のフォーカス遷移で空打ちされ、警告が消費されてしまう。

## 根本原因 / Root cause

- `src/renderer/src/lib/use-canvas-visibility.ts:56-65`: `subscribeOnVisible(cb)` の wrapped 実装が `if (isVisibleNowInternal()) cb()` を即時実行する経路と、event listener 経路の 2 つを併用している。
- `src/renderer/src/lib/use-recruit-listener.ts:135-146`: `pendingHiddenRef.current` に warning を蓄積し、可視化時に flush する設計。
- 短時間 (< 100ms) で hidden → visible → hidden → visible が連続すると、各 visible 通知ごとに `flush()` が呼ばれ、`pendingHiddenRef.current.count = 0` でも空 toast が走る or 中途半端に分割される。
- Windows のタスクバー hover preview や Alt+Tab 中継時に visibilityState がパカパカするため、実環境で頻発する。

## 再現手順 / Repro

1. Canvas モードで起動し、AgentNodeCard を recruit 中にする (warning が出る状況を作る)。
2. Alt+Tab で別アプリに切替 → 即座に Alt+Tab で vibe-editor に戻す、を 200ms 以内に 2-3 回繰り返す。
3. 期待: visibility が安定したタイミングで 1 回だけ警告 toast を出す。
4. 実際: 各遷移で flush() が呼ばれ、警告が割れる or `count == 0` のまま消費判定が立つ。

## 影響範囲 / Impact

- recruit warning を活用するチーム作業ユーザー (Issue #578 のユースケース)
- Windows ユーザー (visibilityState がパカパカしやすい)
- severity: MEDIUM (機能不全ではないが、警告が消える信頼性低下)

## 修正方針 / Fix proposal

### 案1 (推奨): flush を debounce + early return で安定化

1. `subscribeOnVisible` の wrapped cb を debounce (200ms) に変更:
   ```ts
   let timer: number | null = null;
   const debouncedCb = () => {
     if (timer) window.clearTimeout(timer);
     timer = window.setTimeout(() => {
       if (isVisibleNowInternal()) cb();
       timer = null;
     }, 200);
   };
   ```
2. flush 側 (`useRecruitListener` の flush 経路) で `if (pendingHiddenRef.current.count === 0) return;` の早期 return を確実に入れる (現状でも入っているが、closure stale の可能性を確認)。
3. flush 後は `pendingHiddenRef.current = { count: 0, ... }` で必ず reset。reset を debounced 関数の中でアトミックに行う。

### 案2: visibilityState 直接 + idle 判定

`document.visibilityState === 'visible'` かつ `document.hasFocus()` の両方が true の状態が **連続 200ms 以上** 続いたら flush。state machine 化することで race を構造的に排除。
