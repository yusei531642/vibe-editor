TITLE: [security] util: vibe_root() が HOME 不在時に CWD 相対パスへ写真を書き出しうる
LABELS: security,bug,rust,backend

## 概要 / Summary

`vibe_root()` (Rust 側) が `dirs::home_dir().unwrap_or_default()` で home directory を取得しており、HOME 環境変数が無い状況 (= サービスアカウント / sandbox / CI / 環境破損) では空 PathBuf にフォールバックする。この空 PathBuf は **CWD 相対パス** として解釈されるため、paste-images がリポジトリのワーキングディレクトリ配下に書き出され、`cleanup_old_paste_images` がその同名フォルダを 24 時間で勝手に削除する潜在リスクが生じる。

## 根本原因 / Root cause

- `src-tauri/src/util/config_paths.rs:9-11` の `vibe_root()`:
  ```rust
  pub fn vibe_root() -> PathBuf {
      dirs::home_dir().unwrap_or_default().join(".vibe-editor")
  }
  ```
- `unwrap_or_default()` は `PathBuf::new()` (空 path) を返す。`PathBuf::new().join(".vibe-editor")` = `".vibe-editor"` という相対 path。
- `src-tauri/src/commands/terminal/paste_image.rs:93` 周辺の paste-images 書き込みがこの相対 path を base にし、process の CWD = ユーザーのリポジトリ root に対して `paste-images/paste-{uuid}.png` を作成する事故が起きうる。
- `cleanup_old_paste_images` も同じ相対 path を base に走り、リポジトリ内の偶然 `paste-images/` というディレクトリがあれば 24h 経過したファイルを best-effort で削除する。

## 再現手順 / Repro

1. macOS / Linux で `unset HOME` (or HOME=空文字) してから vibe-editor を起動する。
2. ターミナルで画像 paste を発火 (Ctrl+V on xterm)。
3. リポジトリ直下 (CWD) に `paste-images/paste-xxxx.png` が作成される。
4. 同じ環境で 24 時間以上経過すると `cleanup_old_paste_images` が CWD/paste-images/ 配下の古いファイルを削除する。

## 影響範囲 / Impact

- HOME 不在環境 (sandbox / CI / サービスアカウント) のユーザー
- リポジトリに `paste-images/` ディレクトリを意図的に持つユーザー (ドキュメント素材等)
- 攻撃ベクトルは限定的だが、user-controlled ファイルが意図せず削除される **データ破損** の可能性
- severity: MEDIUM (security / bug)

## 修正方針 / Fix proposal

### 案1 (推奨): home_dir() 失敗時に絶対 fallback を使う

```rust
pub fn vibe_root() -> PathBuf {
    if let Some(h) = dirs::home_dir() {
        h.join(".vibe-editor")
    } else {
        std::env::temp_dir().join("vibe-editor")
    }
}
```

`std::env::temp_dir()` は OS の絶対 temp directory を返すため、CWD への fallback を確実に防げる。

### 案2: 起動時に必ず `vibe_root()` を canonicalize し、絶対 path でない場合は panic / 起動拒否

paste-images / settings 書き出しのみで使うので、起動時 1 回 canonicalize し `if !root.is_absolute() { panic!("...") }` で fail-fast。

### 補強: paste-images の write 直前に `vibe_root()` 配下に留まることを確認

```rust
let target = vibe_root().join("paste-images").join(filename);
let canon = target.canonicalize().or_else(|_| Ok(target.clone()));
if !canon.starts_with(&vibe_root()) { return Err(...); }
```

cleanup 経路でも同様にガードする。
